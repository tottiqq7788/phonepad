# 任务：清空所有本地修改，还原到当前最新 Git 版本（纯本地）

## 项目信息

| 项 | 值 |
|---|---|
| 项目名称 | PhonePad |
| 本地路径 | `E:\totti\phonepad\phonepad` |
| 默认分支 | `main` |
| 远程地址 | `git@github.com:tottiqq7788/phonepad.git` |

> 本任务**仅操作本地**，不推送、不拉取、不修改远程仓库。

## 严格执行以下全部流程

1. 不回退 Git 提交历史，保持当前最新版本不变
2. 彻底清空、永久丢弃：所有未提交的修改、新增文件、删除文件、本地手动变更
3. 强制将项目还原到【当前 HEAD 最新提交】，工作区干净
4. 不执行任何远程相关命令（不 fetch / pull / push）
5. 还原完成后自动校验：工作区干净、无变更文件
6. 还原后展示【近期 Git 版本总览】：最近 10 条提交，版本号 + 时间 + 说明，新→旧
7. 输出还原成功结果
8. 总结本次已永久丢弃的所有本地修改内容、文件变动

## 一键执行命令

```powershell
cd E:\totti\phonepad\phonepad

# 先展示即将丢弃的变更（供总结用）
git status
git diff --stat
git diff --cached --stat

# 永久丢弃所有本地修改与未跟踪文件
git reset --hard HEAD
git clean -fd

# 校验工作区
git status
```

## 如果只想丢弃部分变更

```powershell
# 丢弃某个已跟踪文件的修改
git restore 路径/文件

# 取消暂存
git restore --staged .

# 删除未跟踪文件（不删 .gitignore 中的构建产物目录时可改用 -fdx，慎用）
git clean -fd
```

## 还原后版本总览

```powershell
git log --oneline -10
git tag -l --sort=-v:refname | Select-Object -First 10
```

## 注意事项

- `git reset --hard` 和 `git clean -fd` **不可恢复**，执行前必须确认
- 不会删除 `.gitignore` 已忽略的构建目录（如 `node_modules/`、`target/`），除非使用 `git clean -fdx`
- 若本地有 `git stash`，本任务不会自动清除 stash，需另用 `git stash drop`
