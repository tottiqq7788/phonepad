## 必须严格执行以下全部流程

### 1. 进入项目目录并检查 Git 状态

```powershell
cd E:\totti\phonepad\phonepad
```

清晰列出：

- 当前分支
- 是否关联远程分支
- 是否存在未提交代码变更
- 是否存在未跟踪文件

```powershell
git status
git branch -vv
```

### 2. 保护本地未提交变更

如果存在本地未提交变更，**不要直接覆盖或丢弃**：

- 先展示改动文件列表和代码行数变更统计
- 使用 stash 暂存：

```powershell
git stash push -u -m "拉取前自动暂存：$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')"
```

拉取完成后说明本地改动已暂存，可用 `git stash list` 查看，恢复用 `git stash pop`。

### 3. 检查远程更新

```powershell
git fetch origin
git status --short --branch
git log --oneline --decorate --graph --left-right main...origin/main
```

### 4. 拉取最新代码

```powershell
git pull --no-rebase origin main
```

### 5. 拉取完成后展示结果总览

- 当前分支与远程分支同步状态
- 最新提交 hash + 提交说明
- 本次拉取是否有新增提交
- 如果发生冲突，**必须停止**并清晰列出冲突文件，不要擅自丢弃任何改动

### 6. 拉取完成后检查依赖并验证项目

对比本次拉取前后的依赖清单变化，按需安装：

| 变更文件 | 处理命令 |
|---|---|
| `package.json` / `package-lock.json` | 在项目根目录执行 `npm install` |
| `Cargo.toml` / `Cargo.lock` | 执行 `cargo fetch`（或后续构建时自动拉取） |
| `apps/android/**/*.gradle.kts` | 进入 `apps/android` 执行 `.\gradlew.bat --refresh-dependencies` |
