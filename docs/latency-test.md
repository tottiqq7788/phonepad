# 延迟测试

## 目标

在同一 5GHz 局域网下，端到端延迟控制在 **8–16ms** 主观可接受范围。

## 测试步骤

1. 启动桌面接收端，记录本机 IP。
2. Android 连接后，在桌面 UI 观察 **最近 RTT**。
3. 单指缓慢移动，观察鼠标是否跟手。
4. 快速画圆，检查是否有明显拖尾。
5. 单击、双指右键、双指滚动分别测试响应时间。

## 观察指标

| 指标 | 正常范围 | 说明 |
|------|----------|------|
| RTT | < 20ms | 手机到桌面端往返估算 |
| 丢弃包 | 少量可接受 | 旧序列号会被丢弃以保证跟手 |
| 移动包速率 | 60–120/s | 取决于触摸采样率 |

## 优化建议

- 使用 5GHz Wi‑Fi，减少 2.4GHz 干扰
- 关闭手机省电模式
- 桌面端调高灵敏度、适当降低加速度
- 避免 VPN 代理干扰局域网

## 对比 mobile-touchpad

| 项目 | mobile-touchpad | PhonePad |
|------|-----------------|-----------|
| 手机端 | 浏览器 + Hammer.js | 原生 Android |
| 传输 | Socket.IO WebSocket | UDP 二进制 |
| 桌面控制 | robotjs (Node) | enigo (Rust) |
| 反馈 | 无 | 震动 |
| 配对 | 手动输入 IP | 发现 + 二维码 + 手动 |
